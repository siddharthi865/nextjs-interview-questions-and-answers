## Question 1. How do you create a Next.js app without create-next-app?

**Q: How do you create a Next.js app without `create-next-app`?**

**Short Answer (30 seconds):**

You can manually create a Next.js application by initializing a Node.js project, installing the required dependencies (`next`, `react`, and `react-dom`), adding the necessary scripts in `package.json`, and creating the required `app` directory and configuration files. This approach gives you complete control over the project setup and is useful for custom boilerplates or monorepos.

---

# Detailed Explanation

## 1. Initialize the Project

Instead of using `create-next-app`, start with a blank Node.js project.

```bash
mkdir my-next-app
cd my-next-app

npm init -y
```

This creates a basic `package.json`.

---

## 2. Install Dependencies

Install Next.js along with React.

```bash
npm install next react react-dom
```

For TypeScript:

```bash
npm install -D typescript @types/react @types/node
```

On the first run, Next.js automatically generates a `tsconfig.json` if TypeScript files exist.

**Docs:** Next.js → Installation & Project Structure

---

## 3. Configure package.json Scripts

Add the standard scripts:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  }
}
```

These are the same scripts generated by `create-next-app`.

---

## 4. Create the App Router Structure

Create the following folder structure:

```text
my-next-app/
│
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── globals.css
│
├── public/
│
├── package.json
├── next.config.ts
└── tsconfig.json
```

---

## 5. Create the Root Layout

```tsx
// app/layout.tsx

import "./globals.css";

export const metadata = {
  title: "My App",
  description: "Next.js 16 App",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

The root layout is mandatory in the App Router.

---

## 6. Create the Home Page

```tsx
// app/page.tsx

export default function HomePage() {
  return (
    <main>
      <h1>Hello Next.js 16!</h1>
    </main>
  );
}
```

Since this is a **Server Component** by default, no `"use client"` directive is needed.

---

## 7. Add Global CSS

```css
/* app/globals.css */

body {
  margin: 0;
  font-family: sans-serif;
}
```

---

## 8. (Optional) Configure Next.js

```ts
// next.config.ts

import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  reactStrictMode: true,
};

export default nextConfig;
```

Using `next.config.ts` provides type safety and aligns with modern Next.js practices.

---

## 9. Run the Development Server

```bash
npm run dev
```

Visit:

```
http://localhost:3000
```

# Why Create a Project Manually?

This is useful when:

- Building a custom company starter template
- Setting up a monorepo (e.g., Turborepo, Nx, pnpm workspaces)
- Integrating Next.js into an existing project
- Learning the framework internals
- Removing unnecessary starter files
- Creating highly customized project scaffolding

---

# Performance Considerations

- Prefer the **App Router** and **Server Components** for better performance and smaller client bundles.
- Enable `reactStrictMode` during development to catch potential issues early.
- Install only the dependencies you need to keep the project lightweight.
- Organize reusable UI, utilities, and shared logic into dedicated directories (e.g., `components/`, `lib/`) as the application grows.

---

# Common Pitfalls & Solutions

**1. Missing `app/layout.tsx`**

The App Router requires a root layout. Without it, the application will fail to start.

**2. Forgetting scripts**

Without the `dev`, `build`, and `start` scripts, you cannot run or build the application.

**3. Using Pages Router examples**

Most modern Next.js 16 applications should use the App Router unless maintaining an older codebase.

**4. Adding `"use client"` everywhere**

Components are Server Components by default. Only add `"use client"` when browser-only APIs, React hooks (`useState`, `useEffect`), or event handlers are required.

**When to use:**

- Creating internal starter templates
- Monorepo setups
- Migrating an existing Node.js project to Next.js
- Learning how Next.js is structured under the hood
- Projects requiring full control over configuration

**Alternatives:**

- **`create-next-app` (recommended for most projects):** Quickly scaffolds a production-ready project with sensible defaults, TypeScript, ESLint, and optional Tailwind CSS.
- **Manual setup:** Best when you need complete control over the project structure, dependencies, and tooling.
- **Monorepo generators (e.g., Turborepo or Nx):** Ideal for organizations managing multiple applications and shared packages.
